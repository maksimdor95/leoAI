## Value (one-pager)

**LEO Interview Prep** — тренажёр под **вашу вакансию**: разбор → **этапы** в чате → мок → PDF.

**Для кого:** кандидат за несколько дней (или за один вечер) до собеседования.

**Job to be done:**  
Понять, что проверят на *этой* роли, и отрепетировать ответы без хаоса «ещё один чат с вопросами».

| Ценность | Что получает пользователь |
|----------|---------------------------|
| **Контракт** | LEO понял вакансию — кратко, без простыни |
| **Этапы** | Понятная цепочка: диагностика → уроки → практика → мок → готовность |
| **Гибкий темп** | **Спринт** (~3 ч за вечер) или **марафон** (по этапу в день) |
| **Ведение** | Нажал шаг — LEO сам в нужном режиме чата |
| **Выход** | PDF, STAR, вопросы работодателю, уверенность к моку |

**Не обещаем:** «5-дневный курс по календарю» или «выбери режим из шести».

**Обещаем:** «**Следующий шаг** — ~25 мин в чате. Весь маршрут — ~3 ч, если сесть сразу».

**Реализовано в UI (P0):** primary = `PrepNextStepPanel`; разбор и план — accordion; пустые артефакты скрыты; 6 режимов — «вручную».


**Отличие от WannaNew:** пробное собеседование без вакансии vs **маршрут под вакансию**.

## Information Architecture

```
┌──────────────────────────────────────────────────────────────┐
│  СЦЕНАРИЙ           │  ПОДГОТОВКА         │  ЧАТ             │
│  Краткий контракт  │  ★ СЛЕДУЮЩИЙ ШАГ ★  │  Исполнение      │
│  ~X ч · M этапов   │  (клик → режим)     │  текущего этапа  │
│  CTA: Начать       │  Этап N из M        │                  │
│  [▼ Разбор вакансии]│  [▼ Все этапы]      │  треды           │
│                    │  [Артефакты]*       │                  │
│                    │  [Продолжить]*      │                  │
└──────────────────────────────────────────────────────────────┘
  * артефакты — если есть; «Продолжить» — если была пауза
```

### Иерархия (правило v1.4)

1. **Один primary CTA** на экране: «Начать» / «Следующий шаг».
2. **Этап** — единица прогресса; **день** — не показываем как главную метрику.
3. Справочное (разбор, все этапы, курсы) — **свёрнуто**.
4. Режимы (6 кнопок) — **скрыты**, пока есть шаги маршрута.

## Copy-шаблоны (UI)

| Место | Текст |
|-------|-------|

## Чеклист: убрать / свернуть

### Убрать или не показывать до события

- Пустой блок «Артефакты»
- 6 кнопок режимов при активном маршруте этапов
- Прогресс в формате «0/5 **дней**»
- Длинный список условий мока на старте
- Дубль полного разбора на Сценарии и в Подготовке

### Свёрнуто по умолчанию

- Разбор вакансии (6 карточек)
- Все этапы маршрута (бывший «план на 5 дней»)
- Повторная подготовка (только сессия 2+)
- Рекомендуемый календарный темп («1 этап в день, если собес через неделю»)

### Всегда на виду

- Следующий шаг текущего этапа
- Этап N из M + ориентир по времени
- CTA на Сценарии → Подготовка

## North Star (пилот)

**% сессий с завершённым этапом S1 (диагностика) в течение 24 ч после появления контракта S0.**

Дополнительно: **median time-to-S4** (мок) для спринт-сегмента; **D1/D3 return rate** для марафон-сегмента.
