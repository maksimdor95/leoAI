# Руководство по настройке .env файла

## 🎯 Главное понимание

**`env.example`** - это ШАБЛОН (пример)  
**`.env`** - это РЕАЛЬНЫЙ файл с вашими настройками (его нужно создать)

---

## ✅ Что УЖЕ работает без .env

- ✅ Docker контейнеры (PostgreSQL и Redis)
- ✅ База данных PostgreSQL
- ✅ Redis кеш

**Почему?** Потому что в `docker-compose.yml` мы указали значения по умолчанию.

---

## 📝 Что нужно сделать СЕЙЧАС

### Шаг 1: Создать файл .env

```bash
# В PowerShell (в папке проекта)
cp env.example .env
```

**Что произошло:**

- Создался файл `.env` (копия `env.example`)
- Теперь у вас есть реальный файл с настройками

---

### Шаг 2: Открыть .env и заполнить

Откройте файл `.env` в любом текстовом редакторе (VS Code, Notepad и т.д.)

---

## 🔧 Что заполнять СЕЙЧАС (обязательно)

### 1. База данных (PostgreSQL)

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=jack_ai
DB_USER=postgres
DB_PASSWORD=postgres    # ← Можно оставить или изменить
DB_SSL=false
```

**Что делать:**

- Оставьте `postgres` как пароль (если хотите)
- Или измените на свой пароль (запомните его!)

**Пример:**

```env
DB_PASSWORD=мой_секретный_пароль_123
```

---

### 2. Redis

```env
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=          # ← Оставьте пустым (работает без пароля)
REDIS_DB=0
```

**Что делать:**

- Оставьте `REDIS_PASSWORD` пустым (как есть)

---

### 3. Настройки приложения

```env
NODE_ENV=development
APP_NAME=Jack AI Service
APP_VERSION=0.1.0
PORT=3000
```

**Что делать:**

- Оставьте как есть (уже правильно)

---

## ⏸️ Что заполнять ПОЗЖЕ (когда будете разрабатывать)

### AI/NLP Services (OpenAI, Claude)

**Когда:** Когда начнете создавать AI сервис  
**Откуда:** Получите API ключи на сайтах:

- OpenAI: https://platform.openai.com/api-keys
- Anthropic: https://console.anthropic.com/

**Пока:** Оставьте как есть

```env
OPENAI_API_KEY=your_openai_api_key_here
```

---

### Email Service (SendGrid, Mailgun)

**Когда:** Когда начнете создавать Email сервис  
**Откуда:** Зарегистрируетесь на SendGrid или Mailgun

**Пока:** Оставьте как есть

```env
SENDGRID_API_KEY=your_sendgrid_api_key_here
```

---

### Vector Database (Pinecone)

**Когда:** Когда начнете создавать поиск вакансий  
**Откуда:** Зарегистрируетесь на Pinecone

**Пока:** Оставьте как есть

```env
PINECONE_API_KEY=your_pinecone_api_key_here
```

---

### Authentication (JWT)

**Когда:** Когда начнете создавать сервис регистрации  
**Что делать:** Сгенерировать случайные строки

**Пока:** Оставьте как есть (изменим позже)

```env
JWT_SECRET=your_jwt_secret_key_here_change_in_production
```

---

### Job Scraping (LinkedIn, Indeed)

**Когда:** Когда начнете создавать поиск вакансий  
**Откуда:** API ключи от LinkedIn/Indeed (если есть доступ)

**Пока:** Оставьте как есть

```env
LINKEDIN_CLIENT_ID=your_linkedin_client_id
```

---

### Monitoring (Sentry, DataDog)

**Когда:** Когда будете запускать в production  
**Откуда:** Зарегистрируетесь на Sentry/DataDog

**Пока:** Оставьте как есть

```env
SENTRY_DSN=your_sentry_dsn_here
```

---

## 📋 Минимальная настройка СЕЙЧАС

**Минимально нужно заполнить:**

1. ✅ `DB_PASSWORD` - пароль базы данных (можно оставить `postgres`)
2. ✅ Остальное - оставить как есть (значения по умолчанию)

**Пример минимального .env:**

```env
# База данных
DB_PASSWORD=postgres

# Redis (оставить пустым)
REDIS_PASSWORD=

# Остальное оставить как в env.example
```

## 🔒 Важно! Безопасность

### ✅ Что МОЖНО делать:

- Хранить `.env` локально на своем компьютере
- Использовать в разработке

### ❌ Что НЕЛЬЗЯ делать:

- ❌ НЕ коммитить `.env` в Git (он уже в .gitignore)
- ❌ НЕ отправлять `.env` никому
- ❌ НЕ загружать `.env` в публичные места

**Почему?** В `.env` будут храниться секреты (пароли, API ключи)

---

## ✅ Проверка после настройки

После создания `.env`:

1. **Проверьте, что файл создан:**

   ```bash
   # Проверить наличие файла
   ls .env
   # или
   dir .env
   ```

2. **Docker все еще работает?**

   ```bash
   docker-compose ps
   ```

   Должно показать: `Up (healthy)` для обоих контейнеров

3. **Если изменили пароль БД:**
   - Обновите `DB_PASSWORD` в `.env`
   - Перезапустите контейнер: `docker-compose restart postgres`
