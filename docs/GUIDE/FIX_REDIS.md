# Исправление проблемы с Redis

## ✅ Что было исправлено

### Проблема 1: Предупреждение о version

**Было:** `version: '3.8'` в docker-compose.yml  
**Исправлено:** Удалена строка version (она устарела в новых версиях Docker Compose)

### Проблема 2: Redis перезапускался

**Было:** Команда `--requirepass ${REDIS_PASSWORD:-}` требовала пароль, даже если он пустой  
**Исправлено:** Упрощено - Redis работает без пароля по умолчанию

## 🔄 Что нужно сделать сейчас

### Шаг 1: Остановить контейнеры

```bash
docker-compose down
```

### Шаг 2: Запустить заново с исправленной конфигурацией

```bash
docker-compose up -d
```

### Шаг 3: Проверить статус

```bash
docker-compose ps
```

**Должно показать:**

- `jack-postgres` - Up (healthy)
- `jack-redis` - Up (healthy)

### Шаг 4: Проверить Redis

```bash
docker-compose exec redis redis-cli ping
```

**Должно вернуть:** `PONG`

## 📝 Объяснение изменений

### Почему убрали version?

- В новых версиях Docker Compose (v2+) строка `version` больше не нужна
- Она устарела и вызывает предупреждение
- Docker Compose автоматически определяет версию формата

### Почему упростили Redis?

- Redis по умолчанию может работать без пароля (для локальной разработки это нормально)
- Если нужен пароль - можно добавить позже через переменную окружения
- Сейчас главное - чтобы всё работало
