# Настройка Pre-commit Hooks

## Что такое Pre-commit Hooks?

Pre-commit hooks - это автоматическая проверка кода **перед** сохранением в Git. Если код содержит ошибки или не соответствует стилю, сохранение не произойдет.

## Как это работает?

1. Вы изменяете код
2. Вы делаете `git add .` (готовите файлы к сохранению)
3. Вы делаете `git commit` (пытаетесь сохранить)
4. **Автоматически запускается проверка:**
   - ESLint проверяет код на ошибки
   - Prettier форматирует код
   - Если есть ошибки - коммит отменяется
5. Если все хорошо - коммит успешно создается

## Установка (для JavaScript/TypeScript проектов)

### 1. Установить зависимости

```bash
npm install
```

### 2. Инициализировать Husky

```bash
npx husky install
```

### 3. Создать pre-commit hook

```bash
npx husky add .husky/pre-commit "npx lint-staged"
```

### 4. Проверить работу

```bash
# Сделать тестовый коммит
git add .
git commit -m "Test commit"
```

Если есть ошибки - коммит не произойдет, и вы увидите сообщение об ошибках.

## Установка (для Python проектов)

### 1. Установить pre-commit

```bash
pip install pre-commit
```

### 2. Установить hooks

```bash
pre-commit install
```

### 3. Проверить работу

```bash
# Сделать тестовый коммит
git add .
git commit -m "Test commit"
```

## Что проверяется?

### JavaScript/TypeScript файлы

- ESLint - проверка на ошибки
- Prettier - автоматическое форматирование

### Python файлы

- Black - автоматическое форматирование
- isort - сортировка импортов
- flake8 - проверка стиля кода

### Все файлы

- Проверка на trailing whitespace
- Проверка на конфликты слияния
- Проверка JSON/YAML файлов

## Обход проверки (если нужно)

Если нужно срочно закоммитить без проверки (не рекомендуется):

```bash
git commit --no-verify -m "Emergency commit"
```

## Отключение для конкретного коммита

Если нужно пропустить проверку для одного коммита:

```bash
SKIP=lint-staged git commit -m "Skip checks"
```

## Полезные команды

```bash
# Запустить проверку вручную (без коммита)
npx lint-staged

# Проверить все файлы
npm run lint
npm run format:check
```
