# Выбор технологий и структура проекта

## 🎯 Главный вопрос: Какой язык использовать?

### Вариант 1: Node.js (JavaScript/TypeScript)

**Что это:**

- JavaScript - язык для веб-разработки
- Node.js - позволяет запускать JavaScript на сервере
- TypeScript - улучшенная версия JavaScript с проверкой типов

**Плюсы:**

- ✅ Один язык для frontend и backend (JavaScript везде)
- ✅ Быстрая разработка
- ✅ Много готовых библиотек
- ✅ Легко найти разработчиков
- ✅ Хорошо работает с AI API (OpenAI, Claude)

**Минусы:**

- ❌ Может быть медленнее для некоторых задач (но для нашего проекта это не критично)

**Подходит для:**

- WebSocket сервисы (чат)
- REST API (регистрация, авторизация)
- Интеграции с внешними API
- Быстрая разработка MVP

---

### Вариант 2: Python

**Что это:**

- Python - популярный язык для AI/ML и веб-разработки
- FastAPI - современный фреймворк для создания API

**Плюсы:**

- ✅ Отлично для AI/ML (много библиотек)
- ✅ Легко читаемый код
- ✅ Хорошо для парсинга (поиск вакансий)
- ✅ Отлично для работы с данными

**Минусы:**

- ❌ Другой язык для frontend (нужен JavaScript)
- ❌ Может быть медленнее для WebSocket (но решается)

**Подходит для:**

- AI/NLP сервисы (мозг Jack)
- Job Matching (парсинг вакансий)
- Обработка данных

## 🏗️ Структура сервиса (на примере User Profile Service)

### Что будет в сервисе?

```
services/user-profile/
├── src/                    # Основной код
│   ├── controllers/        # Обработчики запросов (регистрация, вход)
│   ├── models/            # Модели данных (User)
│   ├── routes/            # Маршруты API (/api/users/register)
│   ├── middleware/        # Промежуточное ПО (проверка авторизации)
│   ├── services/          # Бизнес-логика
│   └── utils/             # Вспомогательные функции
├── tests/                 # Тесты
├── package.json           # Зависимости (Node.js)
├── Dockerfile             # Для Docker
└── README.md             # Документация
```

### Что делают папки?

**`src/controllers/`** - Обработчики запросов

- Когда пользователь регистрируется → вызывается функция из контроллера
- Аналогия: Как сотрудник на стойке регистрации - принимает запрос, обрабатывает

**`src/models/`** - Модели данных

- Описание структуры пользователя (email, пароль, имя)
- Аналогия: Как форма анкеты - какие поля есть у пользователя

**`src/routes/`** - Маршруты API

- Определяет URL: `/api/users/register` → функция регистрации
- Аналогия: Как адреса - куда отправлять запросы

**`src/middleware/`** - Промежуточное ПО

- Проверка, что пользователь авторизован
- Аналогия: Как охрана - проверяет пропуск перед входом

**`src/services/`** - Бизнес-логика

- Логика работы: проверка пароля, создание пользователя
- Аналогия: Как правила работы - что делать в каждой ситуации

**`src/utils/`** - Вспомогательные функции

- Вспомогательные функции: хеширование пароля, генерация токенов
- Аналогия: Как инструменты - используют везде

## 📊 Сравнение подходов

### Подход 1: Только Node.js

**Плюсы:**

- ✅ Один язык везде
- ✅ Проще для начинающих
- ✅ Быстрее разработка

**Минусы:**

- ❌ AI сервис может быть сложнее (но решается библиотеками)

**Вердикт:** Хорошо для MVP, можно перейти на Python позже для AI

---

### Подход 2: Только Python

**Плюсы:**

- ✅ Отлично для AI
- ✅ Хорошо для парсинга

**Минусы:**

- ❌ WebSocket может быть сложнее
- ❌ Два языка для frontend и backend

**Вердикт:** Хорошо для AI, но сложнее для остального

---

### Подход 3: Гибридный (Node.js + Python)

**Плюсы:**

- ✅ Используем лучшее из обоих миров
- ✅ Node.js для API и WebSocket
- ✅ Python для AI и парсинга
- ✅ Каждый сервис независим

**Минусы:**

- ❌ Нужно знать два языка (но это не проблема, если работаем в команде)

**Вердикт:** ✅ **Лучший вариант для нашего проекта**

---

## 🎯 Рекомендация для MVP

### Фаза 1 (MVP): Начать с Node.js

**Почему:**

- Быстрее разработка
- Проще для начинающих
- Можем создать все сервисы на Node.js
- Потом можно переписать AI сервис на Python, если нужно

**Структура:**

- Все сервисы на Node.js (TypeScript)
- Frontend на React/Next.js
- Позже можем добавить Python для AI

### Фаза 2 (Улучшения): Добавить Python

**Когда:**

- Когда начнем улучшать AI сервис
- Когда начнем парсить вакансии

**Как:**

- Переписываем AI/NLP сервис на Python
- Переписываем Job Matching на Python
- Остальные сервисы остаются на Node.js

## 💬 Вопросы для обсуждения

1. **Какой язык предпочтительнее для вас?**
   - Node.js (JavaScript) - проще для начала
   - Python - лучше для AI (но сложнее для остального)
   - Гибридный - лучшее из обоих

2. **Есть ли предпочтения по команде?**
   - Если есть разработчики на Node.js → используем Node.js
   - Если есть разработчики на Python → можем использовать Python
   - Если есть и те, и те → гибридный подход

3. **Скорость разработки важнее или качество кода?**
   - Скорость → Node.js для всего (MVP быстрее)
   - Качество → Гибридный подход (лучше для каждого сервиса)
