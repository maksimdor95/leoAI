# Code Style Guide

## Автоматическая проверка и форматирование кода

### Для JavaScript/TypeScript

#### ESLint - проверка кода

```bash
# Проверить код на ошибки
npm run lint

# Исправить автоматически исправимые ошибки
npm run lint:fix
```

#### Prettier - форматирование кода

```bash
# Отформатировать весь код
npm run format

# Проверить, нужно ли форматирование
npm run format:check
```

### Для Python

#### Black - форматирование кода

```bash
# Отформатировать код
black .

# Проверить, нужно ли форматирование
black --check .
```

#### isort - сортировка импортов

```bash
# Отсортировать импорты
isort .
```

## Правила форматирования

### JavaScript/TypeScript

- Используем одинарные кавычки (`'`)
- Точка с запятой обязательна
- Ширина строки: 100 символов
- Отступ: 2 пробела

### Python

- Ширина строки: 100 символов
- Отступ: 4 пробела
- Следуем PEP 8

## EditorConfig

Все редакторы кода будут автоматически использовать настройки из `.editorconfig` для:

- Кодировки файлов
- Отступов
- Переводов строк
