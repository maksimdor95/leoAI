# Инструкция по запуску Redis

## Проблема: Redis не запущен

Если вы видите ошибку `ECONNREFUSED` при попытке подключиться к Redis (порт 6379), это означает, что Redis не запущен.

## Решение

### Вариант 1: Запуск Redis через Docker Compose (рекомендуется)

1. **Убедитесь, что Docker Desktop запущен**
   - Откройте Docker Desktop
   - Дождитесь полного запуска (иконка Docker в трее должна быть зеленой)

2. **Запустите Redis через Docker Compose**

   ```bash
   docker-compose up -d redis
   ```

3. **Проверьте, что Redis запущен**

   ```bash
   docker ps
   ```

   Вы должны увидеть контейнер `jack-redis` в статусе "Up".

4. **Перезапустите Conversation Service**
   - Остановите сервис (Ctrl+C)
   - Запустите снова: `npm run dev`

### Вариант 2: Запуск всех сервисов через Docker Compose

Если вы хотите запустить все сервисы (PostgreSQL, Redis) одновременно:

```bash
docker-compose up -d
```

Это запустит:

- PostgreSQL на порту 5432
- Redis на порту 6379
- pgAdmin (опционально) на порту 5050

### Вариант 3: Локальная установка Redis (альтернатива)

Если вы не хотите использовать Docker, вы можете установить Redis локально:

**Windows:**

```powershell
# Используйте WSL2 или установите Redis через Chocolatey
choco install redis-64
```

**macOS:**

```bash
brew install redis
brew services start redis
```

**Linux:**

```bash
sudo apt-get install redis-server
sudo systemctl start redis
```

## Проверка подключения Redis

После запуска Redis вы можете проверить подключение:

```bash
# Если Redis запущен через Docker
docker exec -it jack-redis redis-cli ping

# Должен вернуться: PONG
```

## Устранение проблем

### Проблема: Docker Desktop не запускается

1. Убедитесь, что Docker Desktop установлен
2. Перезапустите Docker Desktop
3. Проверьте, что WSL2 включен (для Windows)

### Проблема: Порт 6379 уже занят

Если порт 6379 уже используется другим приложением:

1. Найдите процесс, использующий порт:

   ```bash
   # Windows
   netstat -ano | findstr :6379

   # macOS/Linux
   lsof -i :6379
   ```

2. Измените порт Redis в `docker-compose.yml`:

   ```yaml
   redis:
     ports:
       - '6380:6379' # Используйте другой порт
   ```

3. Обновите переменную окружения в `.env`:
   ```env
   REDIS_PORT=6380
   ```

### Проблема: Redis контейнер постоянно перезапускается

Проверьте логи контейнера:

```bash
docker logs jack-redis
```

## Автоматическая обработка ошибок

Conversation Service теперь автоматически:

- Повторяет попытки подключения к Redis (до 3 попыток)
- Ждет 5 секунд между попытками
- Логирует ошибки не чаще одного раза в 10 секунд (чтобы избежать спама логов)
- Выводит понятные сообщения об ошибках с инструкциями по исправлению

## Структура Redis в проекте

Redis используется для:

- Хранения активных сессий диалогов
- Кеширования состояния диалога
- Хранения метаданных пользователей

Префиксы ключей:

- `session:{sessionId}` - данные сессии
- `user:{userId}:session` - ID активной сессии пользователя

TTL (время жизни) ключей: 24 часа
