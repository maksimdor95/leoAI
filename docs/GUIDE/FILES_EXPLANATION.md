## 📁 Файлы конфигурации (тоже в основном для чтения)

### 7. `docker-compose.yml`

**Что это:** Файл конфигурации Docker  
**Зачем:** Говорит Docker, какие контейнеры запускать  
**Что делать:**

- **Обычно:** НИЧЕГО! Просто использовать
- **Иногда:** Можем редактировать, если нужно изменить настройки

**Когда редактировать:**

- Если нужно изменить порт (например, 5432 на 5433)
- Если нужно добавить новый сервис
- **Но только после того, как я покажу, как это делать!**

**Пример использования:**

- Вы просто запускаете: `docker-compose up -d`
- Docker читает этот файл и знает, что запускать

---

### 8. `infrastructure/postgres/init/01-init.sql`

**Что это:** SQL скрипт для инициализации базы данных  
**Зачем:** Создает расширения и схемы в базе при первом запуске  
**Что делать:** НИЧЕГО! Он выполняется автоматически  
**Когда выполняется:** Автоматически при первом запуске PostgreSQL

**Важно:** Вы НЕ запускаете его руками! Docker делает это сам.

---

### 9. `.env` (файл, который вы создаете)

**Что это:** Файл с переменными окружения (секреты, настройки)  
**Зачем:** Хранит пароли, ключи API, настройки подключения  
**Что делать:**

- **Создать:** Скопировать `env.example` в `.env`
- **Отредактировать:** Заполнить своими значениями (пароли, ключи)

**Когда редактировать:**

- Когда настраиваете проект впервые
- Когда нужно изменить пароль базы данных
- Когда получаете API ключи

**Пример:**

```bash
# Создать файл из примера
cp env.example .env

# Затем открыть .env и заполнить:
DB_PASSWORD=мой_пароль
OPENAI_API_KEY=мой_ключ
```

## ✅ Что вам нужно делать руками

### 1. При первой настройке:

```bash
# 1. Создать .env файл
cp env.example .env

# 2. Открыть .env и заполнить значения
# (пароли, API ключи)

# 3. Запустить Docker
docker-compose up -d
```

### 2. В процессе работы:

```bash
# Запускать/останавливать контейнеры
docker-compose up -d      # Запустить
docker-compose down       # Остановить
docker-compose ps         # Проверить статус

# Подключаться к базе данных (когда нужно)
docker-compose exec postgres psql -U postgres -d jack_ai
```

### 3. Никогда:

- ❌ Не редактируйте README файлы (они справочники)
- ❌ Не запускайте SQL скрипты руками (они автоматические)
- ❌ Не трогайте Dockerfile'ы (пока не начнем разработку)
