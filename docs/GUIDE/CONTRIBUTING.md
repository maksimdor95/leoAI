# Руководство по вкладу в проект

Спасибо за интерес к проекту Jack AI! Вот как вы можете помочь.

## 🚀 Начало работы

1. **Fork репозиторий** на GitHub
2. **Клонируйте** ваш fork:
   ```bash
   git clone https://github.com/your-username/AIheroes.git
   cd AIheroes
   ```
3. **Создайте ветку** для вашей функции:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## 📝 Процесс разработки

### 1. Следуйте стилю кода

- Прочитайте [CODE_STYLE.md](./docs/CODE_STYLE.md)
- Используйте линтеры и форматтеры перед коммитом
- Код должен проходить все проверки

### 2. Пишите понятные коммиты

Используйте понятные сообщения коммитов:

```bash
# Хорошо
git commit -m "feat: добавить регистрацию пользователей"
git commit -m "fix: исправить ошибку в поиске вакансий"
git commit -m "docs: обновить README"

# Плохо
git commit -m "fix"
git commit -m "update"
```

### 3. Пишите тесты

- Покрывайте новый код тестами
- Убедитесь, что все тесты проходят
- Стремитесь к покрытию >80%

### 4. Обновляйте документацию

- Обновляйте README, если нужно
- Добавляйте комментарии к сложному коду
- Обновляйте API документацию

## 🔄 Pull Request процесс

1. **Убедитесь, что код работает**
   - Все тесты проходят
   - Линтеры не находят ошибок
   - Код отформатирован

2. **Обновите ветку**

   ```bash
   git checkout develop
   git pull origin develop
   git checkout feature/your-feature-name
   git rebase develop
   ```

3. **Создайте Pull Request**
   - Опишите, что изменилось
   - Укажите связанные issues
   - Добавьте скриншоты, если это UI изменения

4. **Ответьте на комментарии**
   - Будьте открыты к фидбеку
   - Вносите изменения, если нужно

## 📋 Чек-лист перед PR

- [ ] Код следует стилю проекта
- [ ] Все тесты проходят
- [ ] Линтеры не находят ошибок
- [ ] Документация обновлена
- [ ] Коммиты имеют понятные сообщения
- [ ] Ветка обновлена с develop

## 🐛 Сообщение об ошибках

Если вы нашли ошибку:

1. Проверьте, нет ли уже issue с этой проблемой
2. Создайте новый issue с описанием:
   - Что произошло
   - Как воспроизвести
   - Ожидаемое поведение
   - Скриншоты, если применимо

## 💡 Предложения

Мы всегда открыты к предложениям! Создайте issue с:

- Описанием предложения
- Обоснованием, почему это полезно
- Примеры использования

## 📞 Вопросы?

Если у вас есть вопросы, создайте issue с меткой `question`.

Спасибо за вклад! 🎉
