# Секреты и переменные деплоя AIheroes

Этот документ содержит полный перечень всех параметров конфигурации, необходимых для успешного деплоя приложения в Yandex Cloud через GitHub Actions.

## 1. Репозиторные секреты (Repository Secrets)

Эти значения должны быть добавлены в GitHub: **Settings -> Secrets and variables -> Actions -> New repository secret**.

| №   | Имя (Name)                    | Описание                  | Где найти в Yandex Cloud                             |
| :-- | :---------------------------- | :------------------------ | :--------------------------------------------------- |
| 1   | `YC_REGISTRY_ID`              | ID реестра Docker-образов | Container Registry -> ID реестра                     |
| 2   | `YC_SERVICE_ACCOUNT_ID`       | ID сервисного аккаунта    | IAM -> Сервисные аккаунты -> ID                      |
| 3   | `YC_SERVICE_ACCOUNT_KEY_JSON` | JSON-ключ для авторизации | IAM -> Аккаунт -> Создать ключ -> Авторизованный     |
| 4   | `YC_ACCESS_KEY_ID`            | ID ключа S3               | IAM -> Аккаунт -> Создать ключ -> Статический        |
| 5   | `YC_SECRET_ACCESS_KEY`        | Секрет ключа S3           | Выдается один раз при создании статического ключа    |
| 6   | `YC_API_KEY`                  | Ключ для Yandex GPT       | IAM -> Аккаунт -> Создать ключ -> API-ключ           |
| 7   | `DB_HOST`                     | Хост PostgreSQL           | Managed PostgreSQL -> Кластер -> Хосты               |
| 8   | `DB_NAME`                     | Имя базы данных           | Managed PostgreSQL -> Кластер -> Базы данных         |
| 9   | `DB_USER`                     | Пользователь БД           | Managed PostgreSQL -> Кластер -> Пользователи        |
| 10  | `DB_PASSWORD`                 | Пароль БД                 | Задается при создании пользователя                   |
| 11  | `DB_PORT`                     | Порт (5432)               | Managed PostgreSQL -> Кластер -> Подключения         |
| 12  | `REDIS_HOST`                  | Хост Redis                | Managed Redis -> Кластер -> Хосты                    |
| 13  | `REDIS_PASSWORD`              | Пароль Redis              | Задается при создании кластера                       |
| 14  | `JWT_SECRET`                  | Ключ безопасности         | Любая случайная строка (придумываете сами)           |
| 15  | `SMTP_HOST`                   | SMTP сервер почты         | Настройки вашей почты (например, smtp.yandex.ru)     |
| 16  | `SMTP_PORT`                   | SMTP порт                 | Обычно 465 (SSL) или 587 (TLS)                       |
| 17  | `SMTP_USER`                   | Логин почты               | Ваш полный email адрес                               |
| 18  | `SMTP_PASSWORD`               | Пароль почты              | Пароль приложения (создается в настройках почты)     |
| 19  | `SMTP_SECURE`                 | Режим SSL/TLS             | true (для порта 465)                                 |
| 20  | `FROM_EMAIL`                  | Email отправителя         | Ваш email                                            |
| 21  | `FROM_NAME`                   | Имя отправителя           | Имя, которое увидят пользователи (например, Jack AI) |

> **Примечание:** В workflow также зарезервированы места под `SENDGRID_API_KEY` и `SENDGRID_FROM_EMAIL`, но они не являются обязательными, если используется SMTP.

## 3. Важные особенности очистки секретов

Все секреты в процессе деплоя проходят автоматическую очистку от лишних символов (пробелы, переносы строк), которые часто попадают при копировании из консоли или блокнота. В workflow используется команда:

```bash
CLEAN_VAR=$(echo "${{ secrets.VAR_NAME }}" | tr -d '\r\n ' )
```

Это гарантирует корректную работу деплоя, даже если в секрете были лишние невидимые символы.
