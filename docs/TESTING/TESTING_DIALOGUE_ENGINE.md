# Руководство по тестированию системы

## Цель тестирования

Проверить, что система работает корректно:

- Вопросы задаются **по одному**, без «простыни»
- Собранные ответы сохраняются в метаданных сессии
- Переходы между шагами происходят последовательно
- AI генерирует дружелюбные формулировки вопросов
- Все агенты (Validator, Profile Analyst, Context Manager) работают корректно
- Новые функции (profile_snapshot, команда профиля, свободное общение) работают правильно

> **Примечание:** Это руководство для **ручного тестирования**. Для автоматических тестов (юнит-тесты) см. [TESTING_INSTRUCTIONS.md](./TESTING_INSTRUCTIONS.md)

### 1. Перезапуск сервисов

```powershell
# Окно 1: Conversation Service
cd C:\Users\Marina\Desktop\AIheroes\services\conversation
Ctrl+C  # остановить, если запущен
npm run dev

# Окно 2: AI/NLP Service
cd C:\Users\Marina\Desktop\AIheroes\services\ai-nlp
Ctrl+C  # остановить, если запущен
npm run dev

# Окно 3: Frontend (если нужно)
cd C:\Users\Marina\Desktop\AIheroes\frontend
npm run dev
```

### 2. Очистка Redis (рекомендуется)

Чтобы убрать старые сессии без новых метаданных:

```powershell
docker-compose exec redis redis-cli FLUSHALL
```

Или очистить только сессии:

```powershell
docker-compose exec redis redis-cli KEYS "session:*" | xargs docker-compose exec redis redis-cli DEL
docker-compose exec redis redis-cli KEYS "user:*:session" | xargs docker-compose exec redis redis-cli DEL
```

### 3. Проверка логов

После запуска сервисов убедись, что видишь:

**Conversation Service:**


### ✅ Тест 3: Последовательность вопросов (Блок 2)

**Шаги:**

1. Ответь на вопрос про должность (например: «HR-менеджер»)
2. Дождись следующего вопроса
3. Ответь на вопрос про опыт (например: «5 лет»)
4. Дождись следующего вопроса
5. Ответь на вопрос про локацию (например: «Москва, не готов к переезду»)

**Ожидаемый результат:**

- ✅ После каждого ответа приходит **только один следующий вопрос**
- ✅ Вопросы идут в правильном порядке:
  1. Должность
  2. Опыт
  3. Локация
  4. Формат занятости
- ✅ Все ответы сохраняются в истории
- ✅ В центре экрана всегда отображается текущий вопрос

**Проверка метаданных:**

После каждого ответа в `metadata.collectedData` должны появляться:

- `desiredRole: "HR-менеджер"`
- `totalExperience: "5 лет"`
- `location: "Москва, не готов к переезду"`
- `workFormat: <твой ответ>` (или `employmentFormat`)

**Как проверить:**

1. **Найти sessionId в логах Conversation Service:**

   ```
   Session created: <sessionId> for user: <userId>
   ```

   Например: `Session created: e374cbcb-e445-430f-af3f-67f4aa1eca57`

2. **Проверить данные в Redis:**

   ```powershell
   # Замени <sessionId> на реальный ID из логов
   docker-compose exec redis redis-cli GET "session:e374cbcb-e445-430f-af3f-67f4aa1eca57"
   ```

3. **Для красивого вывода JSON (опционально):**

   ```powershell
   # Windows PowerShell
   docker-compose exec redis redis-cli GET "session:<sessionId>" | ConvertFrom-Json | ConvertTo-Json -Depth 10

   # Или просто скопировать вывод и вставить в онлайн JSON formatter
   ```

4. **Проверить только collectedData:**
   ```powershell
   # Получить весь JSON и найти collectedData вручную
   docker-compose exec redis redis-cli GET "session:<sessionId>"
   ```

**Что искать в JSON:**

```json
{
  "metadata": {
    "collectedData": {
      "desiredRole": "HR-менеджер",
      "totalExperience": "5 лет",
      "location": "Москва, не готов к переезду",
      "workFormat": "..."
    },
    "currentStepId": "work_format",
    "completedSteps": ["greeting", "role", "experience", "location"],
    "flags": {
      "stepMessageSent:greeting": true,
      "stepMessageSent:role": true,
      ...
    }
  }
}
```

**Альтернативные способы:**

**Способ 1: Через список всех сессий (если не знаешь sessionId):**

```powershell
# Получить все ключи сессий
docker-compose exec redis redis-cli KEYS "session:*"

# Взять первый (или последний) sessionId и проверить
docker-compose exec redis redis-cli GET "session:<sessionId>"
```

**Способ 2: Через браузер DevTools:**

1. Открой DevTools (F12)
2. Перейди на вкладку Network
3. Найди WebSocket соединение (ws://localhost:3002)
4. Посмотри сообщения - найди `session:joined` - там будет `sessionId`

**Способ 3: Через логи Conversation Service:**
В логах ищи строку:

```
Session created: <sessionId> for user: <userId>
```

Скопируй `<sessionId>` и используй в команде Redis.

### ⚠️ Тест 11: Profile Analyst Agent - Пропущены критичные поля (исправлено)

**Шаги:**

1. Пройди диалог, но пропусти критичное поле (например, не ответь на вопрос про должность или ответь "нерелевантно")
2. Дойди до шага `additional` и ответь на него

**Ожидаемый результат:**

- ✅ Profile Analyst обнаруживает пробелы
- ✅ `hasGaps: true`
- ✅ `criticalGaps` содержит отсутствующие поля (например, `['desiredRole']`)
- ✅ Происходит переход на шаг `completion_gap`
- ✅ Вопрос содержит список отсутствующих полей: "Я вижу, что мы не обсудили желаемую должность. Хотите заполнить их сейчас или продолжить как есть?"

**Результат тестирования:**

- ⚠️ **НАЙДЕНА ПРОБЛЕМА:** Profile Analyst правильно обнаружил пробел (`criticalGaps: ["desiredRole"]`), но произошла ошибка `logger.debug is not a function`, из-за чего система перешла в fallback режим и не перешла на `completion_gap`

**Исправления:**

- ✅ Исправлена ошибка `logger.debug is not a function` → заменено на `logger.info()`
- ✅ Profile Analyst теперь корректно возвращает результат анализа
- ✅ Система должна корректно переходить на `completion_gap` при обнаружении пробелов

**Проверка в логах:**

```
Profile analysis for step additional: completeness=0.8, hasGaps=true, readyForMatching=false
  Critical gaps detected: desiredRole
```

**Проверка в Redis:**

- `metadata.currentStepId: "completion_gap"`
- `metadata.flags.completionGapFields: '["desiredRole"]'`

## Известные проблемы и ограничения

### ⚠️ WebSocket предупреждения при перезапуске

**Симптом:** В консоли браузера появляется:

```
WebSocket connection to 'ws://localhost:3002/...' failed: WebSocket is closed before the connection is established.
```

**Причина:** Это нормально при горячей перезагрузке Next.js или перезапуске Conversation Service.

**Решение:** Обнови страницу после того, как все сервисы запустились.

---

### ⚠️ Дубликаты сообщений в истории

**Симптом:** Одно и то же сообщение появляется дважды в истории.

**Причина:** Старая логика отправки сообщений могла оставить дубликаты.

**Решение:** Очисти Redis (`FLUSHALL`) и начни новую сессию.
