# Кейс-тест LEO AI: итоговый отчёт по трём персонам

**Дата:** 17 июня 2026  
**Цель:** профпригодность продукта — матчинг, промпты, UX, retention  
**Персоны:** Иван (аналитик), Андрей (sales), Светлана (психолог)  
**Доп. валидация:** массовая роль «сотрудник склада / водитель» (ручной прогон после отчёта)

---

## 1. Executive summary

LEO **решает задачу для IT-core ролей** с насыщенным каталогом (бизнес-аналитик, sales в Detailed Path). Для **нишевых и non-IT ролей** продукт пока **не даёт честного и понятного результата** — пользователь видит пустую выдачу, чужие вакансии или бесконечный чат без развязки.

**Главный продуктовый паттерн:**

| Путь | Роль | Ценность |
|------|------|----------|
| **Quick Path** (3 вопроса, 1–2 мин) | Hook / intent | Слабый матчинг, часто 0 вакансий — **нормально как тизер**, катастрофа как «финал» |
| **Detailed Path** (5–7 мин) | Core value | Shortlist, оценка профиля, резюме, PDF — **здесь retention** |

**Главный блокер retention (P0):** воронка **Quick → Detailed** не работает как продуктовая механика — переход только «по счастливой случайности», уточнения после Quick уходят в free-chat без UI-развязки.

**Оценка готовности к массовому рынку:** ~70% для IT/fintech/analytics; ~30% для нишевых HR/wellbeing и blue-collar без доработок taxonomy и UX-воронки.

## 2. Методология

- Сценарий Jack: Quick Path + Detailed Path на каждую персону
- Ручной прогон в UI + анализ API (match score, family, catalog)
- Итеративные фиксы в ходе теста (matcher v2, completion screen, профессиональная оценка RQG)
- Scorecard: 5 критериев × шкала 1–5, веса из `TEST_PERSONAS_CASE_STUDY.md`

| Критерий | Вес |
|----------|-----|
| Точность вакансий | 30% |
| Качество диалога | 20% |
| Edge cases / деградация | 15% |
| Скорость до результата | 15% |
| Retention signal | 20% |

## 4. Валидация P0: кейс «сотрудник склада» (ручной прогон)

Повтор кейса после отчёта — **массовая non-IT роль**, не входившая в исходные три персоны.

### Диалог (сокращённо)

1. Quick: роль «Сотрудник склада» → опыт «5 лет в Озоне» → «150 000 в Подольске»
2. Экран: **«✅ Профиль для быстрого подбора готов»** + 2 карточки (Вакансии / Что дальше)
3. Пользователь: **«уточнить детали»** → LEO: «Что уточнить?» → уточнение роли (водитель, доставка…)
4. **6+ раундов** уточнений в чате (локации, формулировки) — **без смены экрана**, без TTS на вопросах
5. Блок «профиль готов» **остаётся висеть** — пользователь не понимает, что диалог продолжается
6. **«перейти к детальному анализу»** — сработало, но только как угадывание фразы → вопрос career_overview

### Диагноз (корень в коде)

| Проблема | Где | Следствие |
|----------|-----|-----------|
| `quick_ready` — терминальный `info_card`, `next: null` | `jackScenario.ts` | UI застывает на «готово», чат живёт отдельно |
| «уточнить детали» → `isQuickReadyMetaClarifyRequest` → **free-chat** | `dialogueEngine.ts` | Нет структуры, нет конца, нет обновления профиля в матчере |
| Нет кнопки **«Детальный анализ»** на экране Quick ready | UI | Переход только текстом, неочевидные формулировки |
| `wantsDetailedProfileAnalysis` не ловит «уточнить детали» | `dialogueEngine.ts` | Путаница intent: уточнить ≠ углубить |
| Free-chat после Quick не меняет `currentStepId` | state machine | Stage panel показывает старый info_card |

### Ожидаемое поведение (продуктовая спецификация)

```
Quick ready screen
├── [Открыть вакансии]     → вкладка Vacancies (даже если 0 — с объяснением)
├── [Углубить профиль]     → Detailed Path, шаг career_overview (явный переход)
└── [Уточнить одно поле]   → мини-форма: роль | опыт | локация | зарплата (max 1–2 шага)
                             → обновить collectedData → rematch → остаться на quick_ready
                             ИЛИ предложить Detailed, если уточнений > 2
```

**Запрещено:** бесконечный free-chat с LEO «уточняющим» без лимита и без смены UI.

## 6. Roadmap: 2–3 спринта

### Спринт 1 — P0 Retention (воронка + честный пустой результат)

**Цель:** пользователь после Quick **всегда** понимает следующий шаг; не теряется на «уточнить детали».

| # | Задача | Критерий готовности | Оценка |
|---|--------|---------------------|--------|
| 1.1 | **Quick ready: кнопка «Углубить профиль»** | Клик → `career_overview`, UI меняется на question mode, TTS | M |
| 1.2 | **Убрать free-chat loop на quick_ready** | «уточнить детали» → меню полей (роль/опыт/локация/ЗП), max 2 итерации | M |
| 1.3 | **Синонимы перехода в Detailed** | «детальный анализ», «углубить», «полный профиль», кнопка | S |
| 1.4 | **Empty match screen** | 0 вакансий → причина + CTA «Углубить» + «Сформировать оценку» | S |
| 1.5 | **Stage panel: не залипать на info_card** | При новом question/command — заменять центральный stage | M |

**Метрика успеха:** ≥80% тестовых пользователей Quick находят переход в Detailed без подсказки; 0 бесконечных clarify-циклов в логах.

## 7. Приоритизация: что делать первым

Если ресурс на **одну неделю** — только блок 1.1–1.5 (воронка Quick → Detailed + UI stage).

Ваш кейс «склад + уточнить детали» — **прямое доказательство**, что P0 важнее новых families: пользователь **хотел углубить**, но попал в болото free-chat, пока экран кричал «готово».

## 8. Продуктовая формула массовости

```
Любая роль
    → LEO понимает family (или честно говорит «ниша»)
    → Quick = hook + явный CTA «Углубить»
    → Detailed = shortlist ИЛИ оценка + резюме + дайджест
```

**Иван** доказывает engine. **Андрей** — что Detailed работает, Quick — нет. **Светлана** — что без taxonomy и честной деградации теряем доверие. **Склад** — что воронка ломается раньше, чем матчинг.

## 10. Рекомендация по P0 (для обсуждения)

**Минимальный shippable fix воронки:**

1. На `quick_ready` — **две primary CTA-кнопки** (не текст в чате):
   - «Открыть вакансии»
   - «Углубить профиль (5 мин)»
2. «Уточнить детали» — **не free-chat**, а `command` с выбором поля → один ответ → `mergeCollectedData` → toast «Профиль обновлён, смотрите Вакансии».
3. После 2 уточнений подряд — soft push: «Для точного подбора лучше пройти детальный анализ» + кнопка.

Готов детализировать техспеку и приступить к реализации 1.1–1.3 по вашему сигналу.
