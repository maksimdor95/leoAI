# Jack: Варианты ветвления сценариев

Ниже — подборка идей, как можно расширить сценарий Jack, чтобы он реагировал гибко и не выглядел линейным. Для каждой ветки указаны цели, условия перехода и пример шагов.

## 1. Приветствие и готовность

- **Цель**: дать пользователю возможность остановиться или перенести разговор.
- **Ветви**:
  - **Готов продолжать** → основной сценарий (`role`).
  - **Нужна пауза** → шаг `pause_reminder`:
    - сообщение Jack: «Понял, я сохраню прогресс. Вернуться сейчас или позже?»
    - команды: `resume_now`, `send_summary`.
    - `resume_now` возвращает на текущий шаг; `send_summary` отправляет собранные факты на email (будущее).
  - **Вопросы по приватности** → шаг `privacy_info`:
    - карточка с объяснением, что данные хранятся локально и в зашифрованном виде.
    - кнопка «Продолжить» → возвращает на приветствие.

## 2. Блок «Желаемая должность»

- **Цель**: адаптировать сценарий под новичков и опытных специалистов.
- **Ветви**:
  - **Опыт < 1 года** → ветка `junior_intro`:
    - другие вопросы: «Учитесь ли сейчас?», «Есть стажировки?»
    - упрощённый список навыков.
  - **Опыт 1–3 года** → основной поток.
  - **Опыт > 10 лет** → ветка `senior_deep_dive`:
    - дополнительные вопросы про управленческий опыт, размер команды, стратегические задачи.

## 3. Локация и формат работы

- **Цель**: учитывать гибкость кандидата.
- **Ветви**:
  - **Готов к переезду** → запросить предпочитаемые города → ветка `relocation`.
  - **Только удалённо** → пропустить вопросы про командировки → сразу перейти к `skills`.
  - **Нужен гибрид** → уточнить количество дней в офисе и расписание → ветка `hybrid_details`.

## 4. Ключевые навыки

- **Цель**: предложить уточнение, если ответ слишком общий.
- **Ветви**:
  - **< 3 навыков** → шаг `skills_clarify` с подсказками («Добавьте инструменты/методологии»).
  - **> 10 навыков** → шаг `skills_prioritize`: «Назовите топ-5, которые важнее всего».
  - **Нет навыков** → шаг `skills_help`, карточка с примерами того, что можно указать.

## 5. Информационный блок (карточки)

- **Цель**: показать промежуточный результат и дать выбор следующих действий.
- **Пример**: после блока «Задачи/Ожидания»
  - карточка `profile_snapshot` с собранными фактами.
  - кнопки:
    - «Продолжить сбор данных» → `education`.
    - «Посмотреть потенциальные вакансии» → ветка `early_matching`.
    - «Отредактировать ответы» → возвращает к нужному шагу (напр. `role`).

## 6. Завершение

- **Цель**: персонализировать завершение.
- **Ветви**:
  - **Профиль полон** → обычная финальная подборка (`matching`).
  - **Пропущены ключевые данные** → шаг `completion_gap`:
    - «Я вижу, что мы не обсудили зарплату и формат работы. Заполнить сейчас или отправить стек как есть?»
    - команды: `fill_missing`, `send_anyway`.
  - **Пользователь просит контакты живого рекрутера** → шаг `escalation`:
    - собрать e-mail/телефон и обещать обратную связь.

## 7. Работа с командами (кнопками)

- **Пауза/возобновление**: кнопка «Пауза» ставит статус `paused`, Jack отправляет «Я сохраняю всё. Вернёмся, когда будете готовы.» Кнопка «Возобновить» снимает паузу.
- **Повторить вопрос**: всегда доступная команда `repeat_question`. Движок повторяет предыдущий вопрос (не создавая новый шаг).
- **Изменить ответ**: команда `edit_last_answer` → Jack уточняет, какой именно ответ заменить, и переходит назад.

## 8. Поведение при непонимании

- Если ответ пустой или совсем нерелевантный, ветка `clarify`:
  - «Кажется, я не уловил суть. Можно коротко, например: …»
  - Переход обратно на тот же шаг, без продвижения вперёд.

## 9. Резюме ветки

Переходы можно реализовать через поле `next`, расширив его до объекта:

```ts
{
  id: 'experience',
  type: 'question',
  next: {
    default: 'location',
    when: [
      { condition: 'experienceYears < 1', to: 'junior_intro' },
      { condition: 'experienceYears > 10', to: 'senior_deep_dive' }
    ]
  }
}
```

- `condition` — выражение над собранными фактами.
- `default` — куда идти, если условия не сработали.

Реализацию можно начинать с 1–2 веток (например, пауза и уточнение навыков), постепенно расширяя сценарий.
