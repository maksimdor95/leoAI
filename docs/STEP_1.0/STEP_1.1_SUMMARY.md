### 3. ✅ Реализована модель User

**Схема базы данных:**

```sql
CREATE TABLE jack.users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

**Файлы:**

- `src/models/User.ts` - интерфейсы TypeScript
- `src/models/userRepository.ts` - методы работы с БД

**Методы:**

- ✅ `createTable()` - создание таблицы
- ✅ `create()` - создание пользователя
- ✅ `findByEmail()` - поиск по email
- ✅ `findById()` - поиск по ID
- ✅ `update()` - обновление пользователя

### 4. ✅ Реализованы API endpoints

#### POST /api/users/register

**Функционал:**

- Регистрация нового пользователя
- Валидация email и пароля
- Хеширование пароля (bcryptjs)
- Проверка уникальности email
- Генерация JWT токена

**Тестирование:**

- ✅ Работает через тестовую страницу
- ✅ Логируется в терминале
- ✅ Сохраняет пользователя в БД

#### POST /api/users/login

**Функционал:**

- Авторизация пользователя
- Проверка email и пароля
- Генерация JWT токена

**Тестирование:**

- ✅ Работает через тестовую страницу
- ✅ Логируется в терминале
- ✅ Возвращает токен

#### GET /api/users/profile

**Функционал:**

- Получение профиля текущего пользователя
- Проверка JWT токена
- Извлечение userId из токена

**Тестирование:**

- ✅ Работает через тестовую страницу
- ✅ Требует авторизацию (токен)
- ✅ Логируется в терминале

#### PUT /api/users/profile

**Функционал:**

- Обновление профиля пользователя
- Проверка JWT токена
- Валидация данных

**Статус:**

- ✅ Реализован
- ⏳ Готов к тестированию
