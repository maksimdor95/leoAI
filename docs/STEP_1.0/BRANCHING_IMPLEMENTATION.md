# Реализация ветвлений в сценарии Jack

## Обзор

Реализована система условных переходов в диалоговом движке, позволяющая создавать нелинейные сценарии на основе ответов пользователя.

## Что реализовано

### 1. Типы и структура

- **Расширены типы сценария** (`services/conversation/src/types/scenario.ts`):
  - Добавлен тип `ScenarioNext` для условных переходов
  - Поддержка структуры `{ default: string, when: Condition[] }`

### 2. Движок диалога

- **Функция `evaluateCondition`** — оценка условий:
  - Числовые сравнения: `<`, `>`, `<=`, `>=` (например, `totalExperience < 1`)
  - Текстовые сравнения: `===`, `!==` с гибким сопоставлением (например, `readyToStart === 'нет'`)
  - Подсчёт элементов: `skillsCount < 3` (автоматически считает навыки через запятую)

- **Функция `resolveNextStep`** — определение следующего шага на основе условий

- **Обработка навыков**: автоматическое объединение дополнительных навыков с основными

### 3. Новые шаги в сценарии

#### Приветствие и готовность

- **`pause_reminder`** — напоминание о паузе (если пользователь написал "нет", "не готов", "позже")
- **`privacy_info`** — информация о приватности (если пользователь спросил про приватность/конфиденциальность)

#### Опыт работы

- **`junior_intro`** — вопросы для новичков (опыт < 1 года)
- **`senior_deep_dive`** — дополнительные вопросы для опытных (опыт > 10 лет)

#### Навыки

- **`skills_clarify`** — уточнение навыков (если указано < 3 навыков)
- **`skills_prioritize`** — приоритизация навыков (если указано > 10 навыков)

## Примеры использования

### Условный переход в сценарии

```typescript
{
  id: 'experience',
  type: 'question',
  label: 'Суммарный опыт',
  collectKey: 'totalExperience',
  next: {
    default: 'location',
    when: [
      { condition: 'totalExperience < 1', to: 'junior_intro' },
      { condition: 'totalExperience > 10', to: 'senior_deep_dive' },
    ],
  },
}
```

### Условия

1. **Числовые сравнения**:
   - `totalExperience < 1` — опыт меньше 1 года
   - `totalExperience > 10` — опыт больше 10 лет

2. **Текстовые сравнения** (с гибким сопоставлением):
   - `readyToStart === 'нет'` — сработает для "нет", "я не готов", "не готов сейчас"
   - `pauseChoice === 'сейчас'` — сработает для "сейчас", "да, сейчас"

3. **Подсчёт элементов**:
   - `skillsCount < 3` — меньше 3 навыков
   - `skillsCount > 10` — больше 10 навыков

## Тестирование

### Тест-кейсы для проверки ветвлений

1. **Приветствие → Пауза**:
   - Ответ: "нет" или "не готов"
   - Ожидаемый результат: переход на шаг `pause_reminder`

2. **Приветствие → Приватность**:
   - Ответ: "приватность" или "конфиденциальность"
   - Ожидаемый результат: переход на шаг `privacy_info`

3. **Опыт → Junior**:
   - Ответ: "0 лет" или "полгода"
   - Ожидаемый результат: переход на шаг `junior_intro`

4. **Опыт → Senior**:
   - Ответ: "12 лет" или "15 лет"
   - Ожидаемый результат: переход на шаг `senior_deep_dive`

5. **Навыки → Уточнение**:
   - Ответ: "Python" (1 навык)
   - Ожидаемый результат: переход на шаг `skills_clarify`

6. **Навыки → Приоритизация**:
   - Ответ: "Python, Java, JavaScript, React, Node.js, SQL, Docker, Kubernetes, AWS, Git, Agile, Scrum" (12 навыков)
   - Ожидаемый результат: переход на шаг `skills_prioritize`

## Мультиагентная система

Диалоговый движок интегрирован с мультиагентной системой для улучшения качества диалога:

### Validator Agent

- Валидирует качество ответов пользователя
- Определяет неясные или нерелевантные ответы
- Переводит на шаг `clarify` при необходимости

### Profile Analyst Agent

- Анализирует полноту профиля при завершении диалога
- Выявляет критичные пробелы
- Переводит на шаг `completion_gap` для заполнения пробелов

### Context Manager Agent

- Отслеживает отклонения от темы диалога
- Мягко возвращает к теме через подсказки
- Сохраняет важную информацию из отклонений

Подробная документация: [MULTI_AGENT_SYSTEM.md](./MULTI_AGENT_SYSTEM.md)

## Новые типы шагов

### info_card

Информационные карточки для отображения структурированной информации:

- `profile_snapshot` — снимок собранного профиля
- `skills_help` — помощь с выбором навыков

### command

Команды/кнопки для управления диалогом:

- `repeat_question` — повторить вопрос
- `edit_last_answer` — изменить предыдущий ответ
- `pause` — поставить диалог на паузу
- `resume` — возобновить диалог
