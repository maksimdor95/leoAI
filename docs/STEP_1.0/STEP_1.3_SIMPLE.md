# Шаг 1.3: Conversation Service (MVP) - Чат с AI

## 🎯 Что мы делаем?

Создаем сервис для общения пользователя с AI в реальном времени:

- WebSocket сервер для чата
- Управление сессиями разговора
- Обработка сообщений пользователя
- Отправка ответов от AI
- Интеграция с AI/NLP сервисом (будет в Шаге 1.4)

**Технологии:** Node.js + Socket.io (WebSocket) + TypeScript  
**Время:** 4-5 дней

---

## 📋 Понимание работы чата (из описания)

### Основные компоненты:

1. **Голосовое взаимодействие**
   - AI говорит голосом (TTS - Text-to-Speech)
   - Транскрибация голоса отображается (зеленый кружок)
   - Пользователь может слушать ответы

2. **Визуальное отображение**
   - Сообщения AI в правой стороне (чат)
   - Вопросы по центру
   - Информационные карточки
   - Команды/кнопки (сценарии)

3. **Управление чатом**
   - Кнопка плей/пауза (остановить/продолжить)
   - Кнопка мьют (выключить/включить звук)
   - Поле ввода для текста

4. **Цель**
   - Собирать информацию о пользователе
   - Рекомендовать вакансии на основе собранной информации

---

## 📋 План действий (по порядку)

### Действие 1: Настроить базовую структуру Conversation Service

**Что делаем:**

- Создать структуру сервиса (Node.js + TypeScript)
- Установить Socket.io для WebSocket
- Настроить базовый Express сервер
- Настроить подключение к Redis (для сессий)

**Время:** 1-2 часа

---

### Действие 2: Реализовать WebSocket сервер

**Что делаем:**

- Настроить Socket.io сервер
- Обработать подключение клиента
- Обработать отправку сообщений
- Обработать отключение клиента

**Время:** 2-3 часа

---

### Действие 3: Реализовать управление сессиями

**Что делаем:**

- Создать модель Session (история разговора)
- Сохранять сессии в Redis
- Загружать историю при подключении
- Управлять жизненным циклом сессии

**Время:** 2-3 часа

---

### Действие 4: Реализовать обработку сообщений

**Что делаем:**

- Принимать сообщения от пользователя
- Сохранять в историю
- Отправлять в AI/NLP сервис (пока заглушка)
- Получать ответ от AI
- Отправлять ответ пользователю

**Время:** 2-3 часа

---

### Действие 5: Реализовать базовые типы сообщений

**Что делаем:**

- Текстовые сообщения
- Вопросы (отображаются по центру)
- Информационные карточки
- Команды/кнопки (сценарии)

**Время:** 2-3 часа

---

### Действие 6: Интегрировать с User Profile Service

**Что делаем:**

- Проверять авторизацию пользователя
- Связывать сессии с пользователем
- Сохранять собранную информацию в профиль

**Время:** 1-2 часа
