# Шаг 1.1: User Profile Service - Простой план

## 🎯 Что мы делаем?

Создаем сервис для работы с пользователями:

- Регистрация (создание аккаунта)
- Авторизация (вход в систему)
- Хранение профилей в базе данных

**Язык:** Node.js (TypeScript)  
**Время:** 3-5 дней

---

## 📋 План действий (по порядку)

### Действие 1: Создать структуру сервиса

**Что делаем:**

- Создать папки для кода
- Настроить package.json
- Настроить TypeScript
- Создать базовую структуру файлов

**Время:** 30-40 минут

---

### Действие 2: Настроить подключение к базе данных

**Что делаем:**

- Подключиться к PostgreSQL
- Настроить библиотеку для работы с БД
- Проверить подключение

**Время:** 20-30 минут

---

### Действие 3: Создать модель User (схема базы данных)

**Что делаем:**

- Определить, какие поля у пользователя (email, пароль, имя)
- Создать таблицу в базе данных
- Настроить миграции

**Время:** 30-40 минут

---

### Действие 4: Реализовать регистрацию

**Что делаем:**

- Создать API endpoint для регистрации
- Хешировать пароль
- Сохранять пользователя в БД
- Проверять, что email уникален

**Время:** 1-2 часа

---

### Действие 5: Реализовать авторизацию

**Что делаем:**

- Создать API endpoint для входа
- Проверять пароль
- Выдавать JWT токен (ключ для доступа)

**Время:** 1-2 часа

---

### Действие 6: Реализовать получение и обновление профиля

**Что делаем:**

- Создать API endpoint для получения профиля
- Создать API endpoint для обновления профиля
- Проверять авторизацию (JWT токен)

**Время:** 1-2 часа

---

### Действие 7: Написать тесты

**Что делаем:**

- Тесты для регистрации
- Тесты для авторизации
- Тесты для получения профиля

**Время:** 1-2 часа

---

## 🎯 Итог Шага 1.1

После выполнения всех действий у нас будет:

- ✅ Работающий сервис регистрации
- ✅ Работающий сервис авторизации
- ✅ API для работы с профилями
- ✅ Подключение к базе данных
- ✅ Тесты

---
