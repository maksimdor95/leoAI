# Шаг 1.4: AI/NLP Service (MVP)

## 🎯 Что делаем?

Создаем сервис, который будет обрабатывать сообщения пользователя и отвечать от лица Jack:

- Получает сообщения из Conversation Service
- Отправляет их в модель (OpenAI/Anthropic)
- Формирует ответ Jack и возвращает в чат
- Извлекает полезную информацию (email, навыки, цели)
- Сохраняет контекст разговора, чтобы помнить, о чем говорили

**Технологии:** Node.js + TypeScript + OpenAI API (или Anthropic) + Redis  
**Время:** 5-7 дней

---

## 📋 План действий

### Действие 1: Настроить основу сервиса

- Создать папку сервиса `services/ai-nlp`
- Настроить TypeScript, зависимости, скрипты
- Создать `.env` и подключить ключ OpenAI
- Настроить подключение к Redis (для контекста)

### Действие 2: Реализовать API для обработки сообщений

- Создать endpoint `POST /api/ai/process-message`
- Принимать сообщение пользователя + сессию
- Загружать контекст (историю) из Redis
- Формировать промпт Jack
- Отправлять запрос в AI модель
- Возвращать ответ + обновленный контекст

### Действие 3: Управление контекстом

- Сохранять историю сообщений (сокращать при необходимости)
- Извлекать важные факты о пользователе (email, навыки)
- Сохранять факты отдельно (structured data)

### Действие 4: Промпты и настройка Jack

- Описать роль Jack (тон, цели)
- Настроить системный промпт
- Настроить примеры (few-shot)
- Настроить фильтрацию ответов

### Действие 5: Rate limiting и безопасность

- Ограничить количество запросов в минуту
- Обрабатывать ошибки (таймауты, лимиты)
- Логировать ответы и ошибки

---

## 📦 Результат

- ✅ Сервис, который принимает сообщения от Conversation Service
- ✅ Возвращает ответы Jack на основе AI модели
- ✅ Сохраняет контекст и факты о пользователе
- ✅ Настроен промпт и стиль общения Jack
- ✅ Ограничение по запросам и обработка ошибок
