### 1.2 Диалог (сбор профиля)

#### Quick Path (3 вопроса):
- **Вопрос 1:** Какую роль ищете? → Естественный, с примером
- **Вопрос 2:** Коротко об опыте → Широкий, но понятный
- **Вопрос 3:** Где работать + зарплата → Объединяет 2 в 1 (эффективно)
- **Время:** ~1.5 минуты → соответствует обещанию

#### Detailed Path (30 вопросов):
- Адаптивная структура: позиция 1 → 7 вопросов, позиция 3 → 3 вопроса
- Вопросы логичные, не раздражают
- Примеры в каждом вопросе помогают

**Проблема:** 30 вопросов — это 5-7 минут, но может ощущаться как «много». Нет прогресс-бара или индикации «осталось 5 вопросов».

### 1.3 Completion Screen (после сбора профиля)

| Элемент | Оценка | Проблема |
|---------|--------|----------|
| Заголовок «✅ Профиль успешно собран!» | ✅ 4/5 | Хорошо |
| Карточки действий (4 шт.) | ⚠️ 2/5 | Дизайн непрезентабельный, тусклый |
| «Профессиональное саммари» | ❌ 1/5 | **Открываешь — оценки НЕТ. Только текст-заглушка.** Нет кнопки генерации. |
| «Резюме» | ✅ 4/5 | Есть кнопка «Сформировать резюме». PDF хороший |
| «Email + сопроводительное» | ⚠️ 3/5 | Работает, но **нельзя сменить email** |
| «Подбор вакансий» | ✅ 4/5 | Просто текст «перейдите во вкладку» — нет кнопки |

#### Детальные проблемы Completion Screen:

**1. Саммари — пустое:**
```
// ProfileCompletionCards.tsx, строка 104-106
<p className="text-xs text-slate-200 leading-relaxed">
  Сводка собирается на основе текущего профиля и помогает быстро адаптировать отклик под вакансию.
</p>
```
Это **статический текст-заглушка**. Нет вызова API генерации саммари. Нет кнопки. Пользователь ожидает увидеть готовое саммари с оценкой профиля, а видит описание фичи.

**2. Дизайн карточек:**
- Тёмные на тёмном фоне → низкий контраст
- Иконки маленькие, текст мелкий (text-xs)
- Accordion-паттерн: открывается одна карточка за раз → пользователь не видит все опции сразу
- Нет визуального CTA (call-to-action) — кнопки спрятаны внутри раскрытой карточки

**3. Email — жёстко привязан к аккаунту:**
```typescript
// services/email/src/controllers/emailController.ts, строка 158
userEmail: user.email  // <- берётся из JWT, менять нельзя
```
Пользователь мог зарегистрироваться через Google (gmail), а хочет получить на рабочую почту. Нет UI для ввода альтернативного адреса.

**4. DOCX — серый фон:**
Проблема скорее всего в шаблоне Puppeteer/docx-генератора — стили не адаптированы под Word. PDF выглядит хорошо, т.к. это прямой рендер.

## ВЫВОД

LEO AI **решает основную задачу** — за 2-6 минут собирает профиль и даёт релевантные вакансии. После внедрения улучшений matcher:
- Scoring стал гранулярным (34–86 вместо 42–62)
- Senior BA Fintech = top-1 (прямое попадание для Ивана)

**Главные блокеры retention:**
1. Пустое «Саммари» — обманутое ожидание
2. Дизайн completion-карточек — не production-ready
3. Email привязка — нет гибкости

**Продукт на ~70% ready** для публичного использования. До идеального сервиса нужно закрыть Priority 1 (4 задачи) — это ~2-3 дня работы.
